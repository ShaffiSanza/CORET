01
Hva er CORET
Tre kjerneprinsipper som definerer alt
■

Struktur over trend

Deterministisk scoringsmotor. Ingen ML i V1. Regler du kan forstå og stole på. Scorer 0–100 basert på kohesjon, proposjoner og plaggroller.

◆

Kontroll over validering

CORET måler — du bestemmer. Ingen "dette er feil"-meldinger. Bare strukturelle muligheter og simulering av endringer.

◊

Engine-first, UI-second

All logikk lever i en ren Swift-pakke (353 tester). UIen er utskiftbar. Motoren er ikke det. Backend (Python/FastAPI) håndterer bilder og API.

06b
Sikkerhet
Strict auth · HSTS · CSP · token isolation · HMAC webhooks
Strict Auth Mode
auth_type per brand: oauth | client_credentials. Ingen fallback. AuthError på feil, aldri tom liste. brand.status = auth_failed | active.
SECURITY
Shopify OAuth
State+nonce+TTL (CSRF). One-time use. Shop domain binding. Scope verification. Preview tokens (brand-scoped, 24h). Rate limit per shop.
SECURITY
Token Management
ENV first, secrets file fallback. Aldri i brands.json. Aldri i API responses. Aldri i logs. Security logger for failed HMAC + invalid keys.
SECURITY
Headers + Validation
HSTS (preload), CSP, XSS, nosniff, DENY. Input max_length + strip(). Webhook HMAC (raw body, timing-safe). Rate limiting (IP + shop).
SECURITY

Fonter: Instrument Serif (titler, score) + DM Sans (body, labels)
Radius: 18px (kort), 10px (sm), 12px (knapper)
Shadows: 3 nivåer (L1 subtle, L2 kort, L3 hero)
Animasjoner: 200–300ms ease-out. Aldri bouncy. Aldri springy.
Frosted glass: backdrop-filter blur(20px) + semi-transparent bakgrunn
Regel: Hvis en skjerm føles høy, travel eller overstimulerende — er den feil.
